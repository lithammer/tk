# Done is terminal in v1

Once a **Ticket** or **Epic** is `done`, v1 of Ticket refuses to transition it back to `active` or `open`. **Start** and **Stop** return exit code 1 with a CLI diagnostic; the **Repository Store** enforces the rule with a schema trigger so any future write path inherits the protection. The store-facing `setItemStatus` helper short-circuits the rejection through a typed `.locked_done` outcome arm rather than relying on raw SQLite errors, keeping user-facing phrasing in `src/messages.zig`. The constraint covers **Item Status** only; title, body, **Priority**, and **Epic** membership remain editable on a `done` item, and **Backend Pull** continues to import current state without going through `setItemStatus`. Resurrection through a dedicated `tk reopen` command is deferred from v1 because the symmetric alternative blurs the intent of `tk done`, would force **Backend Adapters** to invent a "reopen from closed" semantic, and is unnecessary while v1 has no recorded need for it.
